# Matchbox 게이트웨이 기여 가이드 (한국어)

## 1. 목표
본 가이드는 `server/llm/agen_matchbox` 하위 프로젝트에 적용됩니다. 목표는 게이트웨이가 멀티 테넌트,配額(Quota) 통제, API Key 보안, 그리고 에이전트(Agent) 호환성 측면에서 일관된 규격을 유지하도록 보장하는 것입니다.

## 2. 핵심 원칙
- 런타임 시 실제 설정의 진실원은 데이터베이스이며, YAML 파일은 최초 로드 및 백업용으로만 작동합니다.
- `initialize_matchbox(...) -> matchbox() -> get_user_llm(...)` 형태의 단일화된 호출 파이프라인을 훼손해서는 안 됩니다.
-配額(Quota) 계산 귀속인 `sys_paid` (운영자 Key 소모)와 `self_paid` (사용자 Key 소모) 트랙은 명확하게 분리되어 제어되어야 합니다.
- 실제 동작하는 API Key 평문, `.env` 파일, 개인 자격 증명 설정을 Git 저장소에 커밋하는 행위를 엄격히 금지합니다.

## 3. 코드 작성 가이드
- 기능 확장이 필요한 경우 개별 라우터 단에서 커스텀 처리하지 말고, 우선적으로 `manager.py` 및 기능 Mixin 레이어에 공통 하강하여 확장해 주십시오.
- 데이터 모델을 수정할 시에는 메인 프로젝트의 마이그레이션 도구 규격에 맞춰 마이그레이션 스크립트를 파생시키십시오.
- 설정 GUI 화면의 조작 인터페이스와 실제 백엔드 API 명세가 일관되게 매칭되어야 합니다 (플랫폼, 모델, 사용 용도 슬롯, 한도 정책).

## 4. 에이전트 생태계와의 결합
- 본 게이트웨이는 에이전트들의 구동 밑바탕이 되므로, 소스 개작 시 LangChain 및 LangGraph 오케스트레이션 라이브러리와의 호환성에 특히 유의해 주십시오.
- 도구 호출(Function Calling), 스트리밍 토크나이저 출력, 생각 필드(`reasoning`) 통합 보정 처리가 깨지지 않도록 방어해 주십시오.
- 기본 구동 방침을 변경할 시에는 본 저장소 하부의 README 및 코드 템플릿 사용 안내를 동시 갱신해 주십시오.

## 5. 커밋 전 셀프 체크리스트
- API Key 등 개인 보안 정보가 코드 및 파일에 섞여 들어가지 않았는지 재차 확인했습니까.
- 이번 수정이 기존 가입자들의 모델 매핑이나 용도 슬롯 락을 방해하지 않습니까.
-配額(Quota) 차감 판정 및 사용량 로깅 적재 트랙에 예외를 야기하지 않습니까.
- 신규 제어 방식에 대응해 README 등 명세 문서를 적합하게 수정 보완했습니까.
